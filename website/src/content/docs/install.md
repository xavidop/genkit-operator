## Helm chart (recommended)

Each release publishes the chart as an OCI artifact to GHCR. The chart
is stamped with the matching manager image at release time, so no extra
`--set` flags are required.

```bash
helm install genkit-operator \
  oci://ghcr.io/xavidop/charts/genkit-operator \
  --version 0.1.0 \
  --namespace genkit-operator-system --create-namespace
```

Upgrade in place:

```bash
helm upgrade genkit-operator \
  oci://ghcr.io/xavidop/charts/genkit-operator \
  --version 0.2.0 \
  --namespace genkit-operator-system
```

Uninstall:

```bash
helm uninstall genkit-operator --namespace genkit-operator-system
```

## YAML bundle (`install.yaml`)

A consolidated install file is attached to every GitHub Release.

```bash
kubectl apply -f https://github.com/xavidop/genkit-operator/releases/download/v0.1.0/install.yaml
```

This file is generated by `make build-installer IMG=…` and contains
every CRD, the controller `Deployment`, the `Service`, RBAC, and the
namespace — everything needed for a one-shot install.

## From source

Useful when you want to test an unreleased change:

```bash
git clone https://github.com/xavidop/genkit-operator
cd genkit-operator
make install                                # CRDs only
make deploy IMG=ghcr.io/xavidop/genkit-operator:v0.1.0
```

Or against a local [kind](https://kind.sigs.k8s.io/) cluster — this
builds both images locally, loads them into the cluster, and installs
everything in one shot:

```bash
kind create cluster --name genkit
make kind-deploy IMG=genkit-operator:dev
```

## Published images

| Image            | Pull                                                |
| ---------------- | --------------------------------------------------- |
| Manager          | `ghcr.io/xavidop/genkit-operator:<tag>`             |
| Runner (default) | `ghcr.io/xavidop/genkit-runner:<tag>`               |
| Helm chart       | `oci://ghcr.io/xavidop/charts/genkit-operator`      |

Both container images are multi-arch (`linux/amd64` and `linux/arm64`)
and are built as separate per-platform jobs in CI, then merged into a
single manifest list. See the
[release reference](/genkit-operator/reference/release/) for details.
